# Contributing

Thanks for contributing.

## What to contribute

- Improvements to repository-wide instructions
- Better path-specific instruction files
- New reusable skills or agents
- Fixes to examples, scripts, or documentation

## Guidelines

- Keep guidance practical, concise, and easy to reuse.
- Prefer small, focused changes over broad rewrites.
- Keep repository-wide rules in `.github/copilot-instructions.md`.
- Put stack-specific guidance in `.github/instructions/*.instructions.md`.
- Keep skills self-contained under `.github/skills/<skill-name>/`.
- Update `README.md` when the repository structure or usage guidance changes.

## Pull requests

- Explain what changed and why.
- Mention any trade-offs or assumptions.
- Prefer additive changes and avoid unnecessary churn.
